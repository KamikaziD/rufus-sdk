## Unified Migration System for Database Initialization

### Summary

Implements a **unified migration system** where both `rufus db init` (CLI) and `auto_init=True` (SQLite) use the same migration files as a single source of truth. This eliminates schema drift and ensures consistency across all environments.

### Problem

**Before:** Two separate schema sources
- CLI used migration files
- Auto-init used embedded SQL constant
- Risk of schema drift between dev and prod

**After:** Single source of truth
```
migrations/*.sql → MigrationManager → Both CLI and auto-init
```

### Changes

1. **Enhanced MigrationManager** (`tools/migrate.py`)
   - Added connection reuse support
   - Added silent mode for auto-init
   - Fixed SQLite trigger parsing with `executescript()`

2. **Updated SQLitePersistenceProvider** (`src/rufus/implementations/persistence/sqlite.py`)
   - `_create_schema()` now uses MigrationManager
   - Fixed project root path calculation
   - Creates `schema_migrations` table for version tracking

3. **Unified CLI Commands** (`src/rufus_cli/commands/db_cmd.py`)
   - Both SQLite and PostgreSQL use MigrationManager
   - Removed 200+ lines of embedded SQL

4. **New Migration** (`migrations/003_sqlite_fixed.sql`)
   - Simplified schema without complex DEFAULT expressions
   - Application generates IDs (not database)
   - Replaced 002_sqlite_initial.sql (syntax errors)
