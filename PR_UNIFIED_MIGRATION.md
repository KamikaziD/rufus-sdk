# PR: Unified Migration System for Database Initialization

## Overview

Implements a **unified migration system** where all database initialization methods (CLI `rufus db init` and SQLite `auto_init`) use the same migration files as a single source of truth. This eliminates schema drift, simplifies maintenance, and ensures consistency across all environments.

---

## Problem Statement

### Before This PR

**Two separate schema sources:**
1. **CLI (`rufus db init`)**: Used migration files from `migrations/` directory
2. **Auto-init (`SQLitePersistenceProvider`)**: Used embedded SQL constant (`SQLITE_SCHEMA`)

**Issues:**
- ❌ **Schema drift risk** - Two sources of truth could diverge over time
- ❌ **Maintenance burden** - Schema changes required updating two places
- ❌ **Inconsistency** - Dev (auto-init) vs prod (CLI) schemas could differ
- ❌ **No version tracking** - Auto-init didn't record applied migrations
- ❌ **Complex DEFAULT expressions** - Migration 002 had syntax errors

### After This PR

**Single source of truth:**
```
migrations/*.sql (migration files)
           │
    ┌──────┴──────┐
    ▼             ▼
rufus db init  auto_init=True
    │             │
    └─────┬───────┘
          ▼
  MigrationManager
  (applies migrations)
```

**Benefits:**
- ✅ **No schema drift** - Both paths use identical migrations
- ✅ **Single source** - Update migrations once, works everywhere
- ✅ **Version tracking** - `schema_migrations` table for both methods
- ✅ **Simplified maintenance** - One set of files to maintain
- ✅ **Production-ready** - Same schema in dev and prod

## Changes

### 1. Enhanced MigrationManager (`tools/migrate.py`)

**Added connection reuse support:**
```python
def __init__(self, db_url: str = None, migrations_dir: str = "migrations",
             db_type: str = None, conn=None):
    self._owns_connection = conn is None  # Track if we own the connection
    self.conn = conn
```

**Added silent mode:**
```python
async def init_fresh_database(self, silent: bool = False):
    """Initialize fresh database by applying all migrations"""
    await self.init_schema_migrations_table(silent=silent)
    await self.migrate_up(silent=silent)
```

**Fixed SQLite trigger parsing:**
```python
# Before: Split by semicolons (broke triggers)
statements = [s.strip() for s in sql.split(';') if s.strip()]

# After: Use executescript for multi-line statements
await self.conn.executescript(sql)
```

### 2. Updated SQLitePersistenceProvider (`src/rufus/implementations/persistence/sqlite.py`)

**Migration-based schema creation:**
```python
async def _create_schema(self):
    """Create database schema by applying all migrations"""
    from migrate import MigrationManager

    # Correct project root path (5 parents, not 4)
    project_root = Path(__file__).parent.parent.parent.parent.parent
    migrations_dir = project_root / "migrations"

    # Create manager with existing connection
    manager = MigrationManager(
        db_type='sqlite',
        conn=self.conn,
        migrations_dir=str(migrations_dir)
    )

    # Apply all migrations
    await manager.init_fresh_database(silent=True)
```

**Key changes:**
- Uses `MigrationManager` instead of embedded `SQLITE_SCHEMA`
- Fixed project root path calculation (was off by one level)
- Properly handles migration application
- Creates `schema_migrations` table automatically

### 3. Updated Database CLI Commands (`src/rufus_cli/commands/db_cmd.py`)

**Unified initialization for both databases:**
```python
async def _init():
    migrations_dir = Path(__file__).parent.parent.parent.parent / "migrations"

    is_sqlite = db_url.startswith("sqlite")

    if is_sqlite:
        # SQLite: Use MigrationManager
        manager = MigrationManager(db_url, str(migrations_dir))
        await manager.connect()
        try:
            await manager.init_fresh_database()
        finally:
            await manager.close()
    else:
        # PostgreSQL: Use MigrationManager
        manager = MigrationManager(db_url, str(migrations_dir))
        await manager.connect()
        try:
            await manager.init_schema_migrations_table()
            await manager.migrate_up()
        finally:
            await manager.close()
```

**Removed:**
- Embedded schema SQL (200+ lines of SQL code)
- Separate initialization logic

### 4. New Migration File (`migrations/003_sqlite_fixed.sql`)

**Simplified schema without complex DEFAULT expressions:**
```sql
-- Before (002_sqlite_initial.sql):
id TEXT PRIMARY KEY DEFAULT lower(hex(randomblob(16)))  -- Syntax error!

-- After (003_sqlite_fixed.sql):
id TEXT PRIMARY KEY  -- IDs generated by application code
```

**Complete schema:**
- 8 tables (workflow_executions, tasks, compensation_log, etc.)
- 10 indexes for query optimization
- 2 triggers for timestamp management
- All SQLite-compatible syntax

**Migration 002 renamed:**
- `002_sqlite_initial.sql` → `002_sqlite_initial.sql.old` (backup)
