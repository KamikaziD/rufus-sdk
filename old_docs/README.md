## Why These Files Are Archived

These documents served important purposes during development but are now superseded by:
1. **Current implementation** - Features described in plans are now implemented
2. **Consolidated documentation** - Content has been consolidated into user-facing docs
3. **Historical context** - Development process is complete, implementation is stable
4. **Reduced confusion** - Separating historical from current reduces cognitive load

## Accessing Historical Context

If you need to understand:
- **Why a feature was implemented a certain way** → Check planning_docs/
- **When a feature was added** → Check implementation_summaries/
- **What changed in a release** → Check pr_descriptions/
- **Project status at a point in time** → Check status_reports/
